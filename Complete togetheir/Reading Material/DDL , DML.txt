
## üß© SQL Command Categories

SQL commands are grouped into 5 main types:

| Category | Full Form                    | Purpose                                   |
| -------- | ---------------------------- | ----------------------------------------- |
| **DDL**  | Data Definition Language     | Defines the structure (tables, databases) |
| **DML**  | Data Manipulation Language   | Manages the data inside tables            |
| **DCL**  | Data Control Language        | Controls user access/permissions          |
| **TCL**  | Transaction Control Language | Manages transactions (commit/rollback)    |
| **DQL**  | Data Query Language          | Retrieves data (usually `SELECT`)         |

---

## üèóÔ∏è 1. DDL ‚Äî *Data Definition Language*

**Used to define or change the database structure.**

‚û°Ô∏è Affects the schema, not the data.
‚û°Ô∏è Automatically commits (no rollback).

### ‚ú≥Ô∏è Common DDL Commands:

| Command    | Description                        | Example                                          |
| ---------- | ---------------------------------- | ------------------------------------------------ |
| `CREATE`   | Create database or table           | `CREATE TABLE users (id INT, name VARCHAR(50));` |
| `ALTER`    | Modify existing table              | `ALTER TABLE users ADD COLUMN age INT;`          |
| `DROP`     | Delete table or database           | `DROP TABLE users;`                              |
| `TRUNCATE` | Delete all data but keep structure | `TRUNCATE TABLE users;`                          |
| `RENAME`   | Rename table                       | `RENAME TABLE users TO members;`                 |

---

## üßÆ 2. DML ‚Äî *Data Manipulation Language*

**Used to manipulate the data inside tables.**

‚û°Ô∏è Works with rows and columns inside the structure.
‚û°Ô∏è Needs `COMMIT` to make changes permanent (if autocommit = off).

### ‚ú≥Ô∏è Common DML Commands:

| Command  | Description | Example                                    |
| -------- | ----------- | ------------------------------------------ |
| `INSERT` | Add data    | `INSERT INTO users VALUES (1, 'John');`    |
| `UPDATE` | Change data | `UPDATE users SET name='Alex' WHERE id=1;` |
| `DELETE` | Remove data | `DELETE FROM users WHERE id=1;`            |

---

## üß† 3. TCL ‚Äî *Transaction Control Language*

**Used to control transactions** ‚Äî groups of DML operations.

‚û°Ô∏è Ensures data integrity (all or nothing).

### ‚ú≥Ô∏è Common TCL Commands:

| Command             | Description                             | Example                       |
| ------------------- | --------------------------------------- | ----------------------------- |
| `COMMIT`            | Save all changes                        | `COMMIT;`                     |
| `ROLLBACK`          | Undo uncommitted changes                | `ROLLBACK;`                   |
| `SAVEPOINT`         | Create a temporary point to rollback to | `SAVEPOINT sp1;`              |
| `RELEASE SAVEPOINT` | Remove a savepoint                      | `RELEASE SAVEPOINT sp1;`      |
| `SET TRANSACTION`   | Define transaction properties           | `SET TRANSACTION READ WRITE;` |

---

## üîç 4. DQL ‚Äî *Data Query Language*

**Used to retrieve data from tables.**

| Command  | Description | Example                |
| -------- | ----------- | ---------------------- |
| `SELECT` | Get data    | `SELECT * FROM users;` |
